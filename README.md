# Проект: статический сайт на Nginx с CI/CD

## Описание

Проект представляет собой простой статический веб-сайт, развернутый с использованием веб-сервера Nginx в контейнере Docker. Управление контейнерами осуществляется через Docker Compose. Для автоматизации развертывания и обновлений настроен CI/CD пайплайн на базе Drone CI, который отслеживает изменения в репозитории и выполняет перезапуск контейнера с актуальной версией сайта.

## Используемые технологии

- **Nginx** — веб-сервер для обслуживания статических файлов
- **Docker** — контейнеризация приложения
- **Docker Compose** — оркестрация многоконтейнерных приложений
- **Drone CI** — система непрерывной интеграции и доставки

## Запуск проекта вручную

Для локального запуска выполните команду в корневой директории проекта:

```bash
docker compose up -d
```

После запуска сайт будет доступен по адресу: [http://localhost:8080](http://localhost:8080)

Остановка контейнера:

```bash
docker compose down
```

## Как работает CI/CD

Пайплайн Drone активируется при каждом push-коммите в репозиторий. Процесс состоит из одного шага:

1. Остановка текущего контейнера (`docker compose down`)
2. Запуск нового контейнера с обновленными файлами (`docker compose up -d`)

Благодаря монтированию локальной директории в контейнер, изменения в файлах `index.html` и `style.css` применяются сразу после перезапуска без необходимости пересборки Docker-образа.

## Настройка Drone

Для работы CI/CD пайплайна необходимо:

1. Установленный и настроенный Drone CI сервер
2. Подключение Drone к репозиторию (GitHub, GitLab или Bitbucket)
3. В репозитории должен присутствовать файл `.drone.yml` с описанием пайплайна
4. Drone должен иметь доступ к Docker-сокету хоста (`/var/run/docker.sock`) для управления контейнерами

Важно: Drone использует Docker-сокет хоста, поэтому необходимы соответствующие права доступа к `/var/run/docker.sock` на сервере, где запущен Drone.

## Структура проекта

```
.
├── docker-compose.yml    # Конфигурация Docker Compose
├── index.html            # Главная HTML-страница
├── style.css             # Стили для страницы
├── .drone.yml            # Конфигурация CI/CD пайплайна Drone
└── README.md             # Документация проекта
```

### Описание файлов

- **docker-compose.yml**  
  Определяет сервис `web` на основе образа Nginx. Пробрасывает порт 8080 хоста на порт 80 контейнера и монтирует текущую директорию в `/usr/share/nginx/html` для обслуживания статических файлов.

- **index.html**  
  Главная страница сайта с приветственным сообщением. Подключает внешний CSS-файл.

- **style.css**  
  Содержит стили для фона с многоцветным градиентом и центрированного заголовка.

- **.drone.yml**  
  Описание пайплайна Drone типа `docker`. Выполняет остановку и запуск контейнера через команды `docker compose down` и `docker compose up -d`.

## Полезные команды

| Команда | Описание |
|---------|----------|
| `docker compose up -d` | Запуск контейнера в фоновом режиме |
| `docker compose down` | Остановка и удаление контейнера |
| `docker compose logs` | Просмотр логов контейнера |
| `docker compose ps` | Проверка статуса контейнера |
| `curl http://localhost:8080` | Проверка доступности сайта |

## Примечание

Перед запуском убедитесь, что порт 8080 на хосте не занят другим процессом. При необходимости измените порт в файле `docker-compose.yml`.
